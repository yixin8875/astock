# 字段填写规范（实操版）

## 1. 题材定义

- `theme_name`：短词，避免同义词重复（如“AI算力”“算力基础设施”二选一）
- `theme_type`：
  - `concept`：市场概念
  - `policy`：政策驱动
  - `cycle`：周期驱动
- `lifecycle_stage`：`early/growth/mature/decline`

## 2. 公司-题材相关性（`relevance_score`）

- `90-100`：主营业务核心受益，业绩弹性明确
- `70-89`：业务重要受益，但非唯一驱动
- `50-69`：边际受益，验证项不足
- `<50`：弱关联，仅情绪映射

## 3. 投资逻辑写法（`thesis_body`）

建议固定三段：

1. 驱动因子：需求/政策/供给变化
2. 传导路径：为什么这家公司受益
3. 验证指标：未来 1-4 个季度如何验证

## 4. 催化（`catalysts`）

- 要求可落到时间点（财报、政策窗口、订单公告）
- 每条催化写明方向与级别

## 5. 风险（`risks`）

- 风险必须可触发，避免“空泛风险”
- 同时写概率和严重度，便于后续做风险评分

## 6. 来源（`sources`）

- 优先级：公告 > 财报会纪要 > 一手调研 > 新闻
- `credibility_score` 建议：
  - 公告/年报：80-100
  - 主流券商深度：60-80
  - 一般新闻：30-60
